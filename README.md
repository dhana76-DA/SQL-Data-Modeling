# SQL-Data-Modeling

1. Objective
To design and implement a Star Schema data model for the Global Superstore dataset to support efficient analytical queries, reporting, and business intelligence operations.

Why Star Schema?
A Star Schema:
Improves query performance
Simplifies data analysis
Is ideal for OLAP workloads
Enables fast aggregation and slicing/dicing

To design and implement a Star Schema using SQL for the Global Superstore dataset.
Tools Used:
MySQL, dbdiagram.io

Procedure:
Studied dataset attributes.
Identified fact and dimension tables.
Designed star schema.
Created ER diagram using dbdiagram.io.
Implemented schema using SQL.
Executed analytical queries.

Result:
A fully functional star schema was implemented and tested using SQL queries.
